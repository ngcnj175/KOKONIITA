# ココニイタ。(KOKONIITA)

現実世界の場所に「記憶」を置くWebサービス。

## 現状のプロトタイプ

### 実装済み
- **レーダー画面**（自分＝中心、記憶＝光点、方位に合わせて回転）
  - 表示範囲切替：100m / 500m / 1km / 5km（ボタンで循環）
  - 近い記憶が重なる場合はクラスタリング（大きな点＋件数の数字）
  - 圏外の記憶は縁に方向インジケータとして表示
  - スイープ光のアニメーション
- **アップロード画面**
  - 写真の選択・自動圧縮（長辺1024px / JPEG 60%）
  - ポラロイド枠プレビュー
  - GPS精度ゲート：**精度が±20mを超える間は「ここに置く」ボタンをグレーアウト**
  - タイトル・ひとこと入力
- **記憶詳細**
  - 20m以内で解放されるポラロイド表示
  - 遠い時は「距離: 約○m」のみ表示
  - 「回収する」（削除）

### 実装済み（バックエンド）
- Cloudflare Workers + D1 + R2 + KV による API
- Google OAuth ログイン（投稿・履歴・回収時に必須）
- 全ユーザーの記憶をレーダーに表示、履歴は自分のみ
- 3種類の公開範囲: **🌐 全体 / 🔒 自分だけ / 👥 合言葉**
  - 合言葉モードは投稿時に6桁の合言葉を発行（空欄なら自動、任意入力で好きな合言葉も可）。伝えた相手だけがレーダー画面上部のモード切替から「👥」を選び、合言葉を入れると光点として見える（20mまで近づくと写真解放）
  - 同じ合言葉に複数投稿を紐付けできる（例: 町の宝探しイベントで1キー = 複数スポット）。合言葉の所有権は最初に使ったユーザーに固定される
- セットアップ手順は [backend/README.md](backend/README.md)

### 未実装（次フェーズ）

- geohashによる近傍クエリ最適化

## 動かし方

### PCで確認
```powershell
python -m http.server 8000
# → http://localhost:8000
```
※HTTPSまたはlocalhostでないと Geolocation API が動きません。

### スマホ実機で確認（推奨）
GitHub Pages にデプロイして実機テストするのが最短です。

1. このディレクトリを GitHub リポジトリにpush
2. リポジトリ Settings → Pages → Source を `main` ブランチに設定
3. 数分後に `https://<username>.github.io/KOKONIITA/` が公開される
4. スマホで開き、位置情報と方位センサーの許可を与える

## ファイル構成
- `index.html` — 画面構造（レーダー / アップロード / 詳細モーダル）
- `style.css` — 世界観（ダーク＋ポラロイド＋琥珀色のアクセント）
- `app.js` — ロジック（GPS、方位、レーダー描画、投稿、localStorage）

データは現段階では localStorage のみ。ブラウザを変えるとデータは見えません。
